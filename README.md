## üåü About SHEBnks

**SHEBnks** is a revolutionary financial technology platform designed specifically for **women and women-owned Micro, Small, and Medium Enterprises (MSMEs)** in Africa. We provide a comprehensive, one-stop mobile application that breaks down traditional banking barriers and creates accessible pathways to financial independence.

### The Problem We're Solving

- **Financial Exclusion**: Millions of African women lack access to formal financial services
- **Limited Credit Access**: Women-owned businesses struggle to secure loans from traditional banks
- **Financial Literacy Gap**: Many women lack the knowledge to make informed financial decisions
- **High Transaction Costs**: Traditional banking services are expensive and inaccessible
- **Fragmented Services**: Women must use multiple platforms for different financial needs

### Our Solution

SHEBnks consolidates **6 essential financial services** into a single, user-friendly mobile application:

1. **SHELoans** - Access to friendly, low-interest loans
2. **SHEFunds** - Seed funding for STEM-related businesses
3. **SHEInsures** - Comprehensive health insurance services
4. **SHELearns** - Financial education and literacy programs
5. **SHEShops** - E-commerce platform for women artisans
6. **SHEiQ** - AI-powered financial coaching and insights

## üöÄ Getting Started

### Prerequisites

**For Mobile Development**:
- Node.js (v16+) or Dart SDK
- React Native CLI or Flutter SDK
- Android Studio (for Android development)
- Xcode (for iOS development - macOS only)
- Java JDK 11+
- CocoaPods (for iOS)

**For Backend Development**:
- Node.js (v16+) or Python (3.9+)
- PostgreSQL (v13+)
- MongoDB (v5+)
- Redis (v6+)
- Docker & Docker Compose

**For Back Office Development**:
- Node.js (v16+)
- Modern browser (Chrome, Firefox, Edge, Safari)

### Installation

```bash
# Clone the repository
git clone https://github.com/dotconnectafrica/shebnks.git
cd shebnks

# Install dependencies for mobile app
cd mobile
npm install  # or: flutter pub get

# Install dependencies for backend
cd ../backend
npm install  # or: pip install -r requirements.txt

# Install dependencies for back office
cd ../back-office
npm install

# Set up environment variables
cp .env.example .env
# Edit .env with your configuration

# Set up database
npm run db:migrate  # or: python manage.py migrate

# Seed initial data (optional)
npm run db:seed

# Start development servers
npm run dev  # or: docker-compose up
```

### Running the Applications

**Mobile App**:
```bash
cd mobile

# For Android
npm run android  # or: flutter run

# For iOS (macOS only)
npm run ios  # or: flutter run
```

**Backend API**:
```bash
cd backend
npm run dev  # or: python manage.py runserver
# API will be available at http://localhost:3000
```

**Back Office Dashboard**:
```bash
cd back-office
npm start
# Dashboard will be available at http://localhost:3001
```

### Testing

```bash
# Run unit tests
npm run test

# Run integration tests
npm run test:integration

# Run e2e tests
npm run test:e2e

# Generate coverage report
npm run test:coverage
```

## üîê Security

### Security Measures Implemented

**1. Data Encryption**:
- TLS 1.3 for data in transit
- AES-256 encryption for data at rest
- Encrypted database backups
- Secure document storage

**2. Authentication & Authorization**:
- OAuth 2.0 and JWT tokens
- Two-factor authentication (2FA)
- Biometric authentication (mobile)
- Role-based access control (RBAC)
- Session management with auto-logout

**3. API Security**:
- Rate limiting to prevent abuse
- CORS configuration
- API key rotation
- Input validation and sanitization
- SQL injection prevention
- XSS and CSRF protection

**4. Compliance**:
- GDPR compliance
- PCI DSS compliance (payments)
- KYC/AML procedures
- Data privacy regulations
- Regular security audits

**5. Back Office Security**:
- IP address whitelisting
- DCA email domain restriction
- Mandatory 2FA for all users
- Audit logging (2-year retention)
- Password policies and complexity requirements

### Security Best Practices

```javascript
// Example: Secure API call with JWT
const apiClient = axios.create({
  baseURL: process.env.API_URL,
  timeout: 10000,
  headers: {
    'Content-Type': 'application/json',
  },
});

// Add JWT token to requests
apiClient.interceptors.request.use((config) => {
  const token = getSecureToken();
  if (token) {
    config.headers.Authorization = `Bearer ${token}`;
  }
  return config;
});

// Handle token expiration
apiClient.interceptors.response.use(
  (response) => response,
  async (error) => {
    if (error.response?.status === 401) {
      await refreshToken();
      return apiClient.request(error.config);
    }
    return Promise.reject(error);
  }
);
```


```http
POST   /loans/apply          # Submit loan application

GET    /loans/:id/schedule   # Get repayment schedule
```

## ü§ù Contributing

We welcome contributions from the development community! Here's how you can help:

### Getting Started

1. **Fork the repository**
2. **Clone your fork**: `git clone https://github.com/your-username/shebnks.git`
3. **Create a branch**: `git checkout -b feature/your-feature-name`
4. **Make your changes**
5. **Write/update tests**
6. **Commit your changes**: `git commit -m "Add: your feature description"`
7. **Push to your fork**: `git push origin feature/your-feature-name`
8. **Submit a pull request**

### Coding Standards

**JavaScript/TypeScript**:
- Use ESLint and Prettier
- Follow Airbnb Style Guide
- Write JSDoc comments
- Use meaningful variable names
- Keep functions small and focused

**Python**:
- Follow PEP 8 style guide
- Use type hints
- Write docstrings
- Use Black for formatting

**Git Commit Messages**:
- Use conventional commits format
- Examples:
  - `feat: add loan calculator component`
  - `fix: resolve payment gateway timeout`
  - `docs: update API documentation`
  - `test: add unit tests for auth service`
  - `refactor: optimize database queries`

### Code Review Process

1. All code must be reviewed by at least one team member
2. All tests must pass
3. Code coverage must not decrease
4. Documentation must be updated
5. No merge conflicts

### Testing Requirements

- Unit test coverage: minimum 80%
- Write integration tests for API endpoints
- E2E tests for critical user flows
- Test on both Android and iOS (mobile)
- Test on multiple browsers (web)
